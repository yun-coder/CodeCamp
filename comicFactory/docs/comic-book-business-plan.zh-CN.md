# Comic Factory 彩色漫画书商业方案

## 1. 产品定位

Comic Factory 从「HTML 生成视频」升级为「输入想法生成漫画小说图片集合」的 AI 创作工作台。

目标不是做一个普通的 AI 出图网站，而是交付一套可编辑、可导出、可连载的漫画小说图片集合：

- 输入：一句想法、小说片段、文章链接、角色设定、品牌资料。
- 输出：封面、角色卡、章节画面清单、彩色漫画页、对白气泡、PDF、图片包、Webtoon 长图。
- 核心价值：让没有绘画团队的创作者和内容团队，也能在可控成本内做出完整漫画作品。

一句话定位：

> 面向独立创作者、教育内容团队、短剧/小说 IP 团队的 AI 漫画小说图片集合生成工具。

## 2. 市场判断

漫画市场本身仍在增长，数字漫画和移动阅读扩大了创作者触达读者的渠道。AI 漫画生成器已经出现一批竞品，说明需求成立，但也说明单纯「输入 prompt 出图」很难形成壁垒。

同类方向包括 ComicForge、ComicInk、Anifusion、MangaOra 等。这些产品共同证明了三个用户刚需：

1. 用户想要完整故事，而不是孤立图片。
2. 用户最痛的是角色一致性、风格一致性和分镜排版。
3. 用户需要导出、编辑、发布，而不是一次性生成。

所以 Comic Factory 的差异化应放在「成书工作流」和「可编辑生产流」，而不是泛化图片生成。

## 3. 首批目标客户

优先避开对 AI 抵触较强的职业漫画出版链条，先服务更容易付费和验证的轻量生产场景。

- 小红书、抖音、B 站故事号创作者：需要连续漫画内容、竖版长图和可拆分发布的图片集合。
- 网文作者：把小说开篇、人物关系、名场面转成漫画，用于引流。
- 教培机构：把知识点、历史故事、安全教育做成漫画课件。
- 企业品牌与营销团队：把产品案例、客户故事、品牌介绍做成漫画。
- 家庭与儿童内容用户：生成个性化儿童故事书、生日礼物书。

## 4. MVP 范围

第一版只做「8 组彩色漫画小说图片集合」，不做长篇连载平台。

必须包含：

- 一句话生成故事大纲。
- 自动生成角色圣经，包括外貌、服装、表情、色板、角色参考图提示词。
- 自动拆成 8 组画面，每组 2-6 张图片。
- 三种风格：美式彩漫、儿童绘本、Webtoon。
- 页面中对白与旁白使用 HTML/CSS 后贴，避免图片中文字乱码。
- 支持单页或单格重绘。
- 导出 PDF、PNG 图片包、Webtoon 长图。

暂不做：

- 多人实时协作。
- 在线漫画社区。
- 100 页长篇自动生成。
- 复杂出版排版。
- 训练专属模型。

## 5. 产品工作流

核心流水线：

```text
Idea
  -> Story Brief
  -> Character Bible
  -> Chapter / Page Plan
  -> Panel Script
  -> Image Prompt Pack
  -> Panel Images
  -> Lettering Overlay
  -> Comic Page HTML
  -> PDF / PNG / Webtoon Export
```

现有项目已经具备的能力：

- 多帧内容图：可改造为漫画页与分镜结构。
- HTML/CSS/GSAP 渲染：可用于漫画页面排版、气泡、标题和动态预告。
- Studio UI：可改造成漫画编辑器。
- 视频与音频生成：作为遗留能力保留，不进入漫画小说图片集合的主流程。

需要新增的能力：

- ComicBook IR：故事、角色、页面、分镜、对白、导出格式。
- 角色一致性系统：角色卡、参考图、锁定描述、重绘约束。
- 图像模型适配器：文本/参考图到分镜图。
- PDF/PNG/Webtoon 导出。
- 版权与安全策略：禁止知名 IP 角色、在世艺术家风格仿制、侵权品牌形象。

## 6. 商业模式

建议采用 SaaS + 点数制。

| 版本 | 价格建议 | 适合人群 | 权益 |
| --- | --- | --- | --- |
| Free | 免费 | 试用用户 | 每月 1 本 4 页低清，有水印 |
| Creator | 39-79 元/月或 $19/月 | 个人创作者 | 8-16 页生成、高清导出、基础角色库 |
| Pro | 199 元/月或 $49/月 | 内容团队 | 商用导出、角色库、批量重绘、无水印 |
| Studio | 3000-20000 元/年或 $199/月起 | 教育/品牌/工作室 | 团队空间、品牌风格包、API、私有部署 |

成本控制原则：

- 首图生成前先完成文本结构和页面规划，避免盲目出图。
- 只对分镜主体出图，文字和气泡后贴。
- 支持局部重绘，不默认整页重绘。
- 对免费用户限制分辨率、页数和并发。

## 7. 获客策略

前 90 天只做样书驱动。

1. 制作 20 本垂直样书：儿童安全、职场故事、AI 科普、品牌案例、网文开篇、产品说明。
2. 每本样书同时导出 PDF、PNG 图片包、Webtoon 长图。
3. 在小红书/抖音/B 站发布「输入一句话生成一本漫画书」过程视频。
4. 找 20 位网文作者和知识博主免费试用，换案例与公开反馈。
5. 为每本公开样书生成分享页，沉淀 SEO 和模板入口。

## 8. 关键指标

- 首次输入到可读漫画完成率：大于 40%。
- 8 页漫画首次生成耗时：小于 5 分钟。
- 单本 8 页生成成本：控制在 1-3 元人民币，视模型而定。
- 单页重绘率：小于 30%。
- 免费到付费转化：大于 3%。
- 30 日留存：大于 20%。

## 9. 风险与应对

| 风险 | 表现 | 应对 |
| --- | --- | --- |
| 角色漂移 | 每页角色长得不一样 | 角色圣经、参考图、角色锁定提示词、分镜前一致性检查 |
| 文字乱码 | 图片中文字不可读 | 图像只生成画面，文字用 HTML/CSS 气泡后贴 |
| 风格不统一 | 每页像不同作品 | 风格包、色板、页面模板、统一 prompt pack |
| 成本失控 | 用户反复重绘整页 | 点数制、局部重绘、先草稿后高清 |
| 版权风险 | 生成知名 IP 或仿名家 | 敏感词过滤、原创风格包、版权提示、AI 使用记录 |
| 质量不可控 | 第一版不能直接发布 | 提供编辑器和重绘入口，不承诺一次完美 |

## 10. 6-8 周落地计划

### 第 1-2 周：结构层改造

- 新增 ComicBook IR。
- Project 挂载 comicBookPath、comic settings。
- 增加漫画书计划写入/读取接口。
- 更新 README 与产品定位。

### 第 3-4 周：生成链路

- 将用户 idea 生成 story brief、character bible、page plan。
- 生成 page/panel prompt pack。
- 打通一个图像模型适配器。
- 先生成 4-8 页低清草稿。

### 第 5-6 周：编辑器与导出

- Studio 改成漫画项目视图。
- 增加页面缩略图、分镜文本编辑、单格重绘。
- 支持 PNG 页面包和 PDF 导出。

### 第 7-8 周：商业验证

- 完成 20 本样书。
- 增加水印、点数、导出限制。
- 上线公开分享页。
- 邀请首批创作者试用。

## 11. 当前仓库第一阶段改造目标

本阶段的代码改造只做基础设施：

- 保留现有视频生成能力作为兼容能力，但默认工作流转向图片集合。
- 新增漫画书 IR，不破坏原有 ContentGraph。
- 给 Project 增加漫画书计划字段。
- 提供写入/读取漫画书计划的 orchestrator 方法。
- 后续再接 UI、图像模型和 PDF 导出。
