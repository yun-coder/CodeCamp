## 核心概念

### 属性系统架构

```
window.HIPRINT_CONFIG (全局配置)
    ↓
hiprint.init() (初始化)
    ↓
PrintTemplate (模板)
    ↓
settingContainer (属性容器)
    ↓
PrintElementOptionSetting (属性面板)
```

### 关键对象

1. **`window.HIPRINT_CONFIG`**: 全局配置对象，定义可用的属性项
2. **`settingContainer`**: 属性面板的 DOM 容器选择器
3. **`options`**: 元素实例上的属性对象
4. **`optionItems`**: 属性项定义数组

## 快速开始

### 最简单的例子（5 分钟上手）

```javascript
// 1. 定义自定义属性
window.HIPRINT_CONFIG = {
  optionItems: [
    {
      name: 'myCustomProp',
      title: '我的属性',
      type: 'input',
      defaultValue: '默认值'
    }
  ]
};

// 2. 初始化 hiprint
window.hiprint.init({
  providers: [new customElementTypeProvider()]
});

// 3. 创建模板并绑定属性容器
const template = new window.hiprint.PrintTemplate({
  template: {},
  settingContainer: '#PrintElementOptionSetting'  // 属性显示在这里
});

template.design('#hiprint-printTemplate');

// 4. 在元素中使用自定义属性
{
  tid: 'test.element',
  text: '测试元素',
  type: 'text',
  options: {
    myCustomProp: '我的值',  // 使用自定义属性
    width: 100,
    height: 30
  }
}
```

## 5 种实现方式

### 方式 1：全局配置（推荐 ⭐⭐⭐⭐⭐）

**适用场景**: 需要在多个元素类型之间共享属性

```javascript
window.HIPRINT_CONFIG = {
  optionItems: [
    {
      name: 'customColor',
      title: '自定义颜色',
      type: 'color',
      defaultValue: '#1890ff',
      onChange: function(value, element) {
        console.log('颜色改变:', value);
      }
    }
  ],
  
  // 指定哪些元素类型支持这个属性
  text: {
    supportOptions: [
      { name: 'width' },
      { name: 'height' },
      { name: 'customColor' }  // 添加自定义属性
    ]
  }
};
```

**优点**:
- ✅ 属性可复用
- ✅ 统一管理
- ✅ 自动显示在属性面板

**缺点**:
- ❌ 需要在初始化前配置

---

### 方式 2：元素定义时配置（推荐 ⭐⭐⭐⭐）

**适用场景**: 元素特有的属性

```javascript
{
  tid: 'myModule.specialElement',
  text: '特殊元素',
  type: 'text',
  options: {
    // 标准属性
    width: 100,
    height: 30,
    
    // 自定义属性
    specialProp: 'value',
    
    // 自定义格式化
    formatter: function(data, options) {
      return options.specialProp + ': ' + data;
    }
  }
}
```

**优点**:
- ✅ 直接在元素定义中
- ✅ 不需要额外配置
- ✅ 灵活性高

**缺点**:
- ❌ 不会自动显示在属性面板（需要配合方式 1）
- ❌ 属性不可跨元素复用

---

### 方式 3：动态扩展（推荐 ⭐⭐⭐）

**适用场景**: 运行时动态添加属性

```javascript
// 在模板初始化后
const element = hiprintTemplate.getElementByField('myField');
if (element) {
  // 动态添加属性
  element.options.dynamicProp = 'dynamic value';
  element.refresh();
}
```

**优点**:
- ✅ 运行时灵活
- ✅ 可以根据条件添加

**缺点**:
- ❌ 不会显示在属性面板
- ❌ 需要手动管理

---

### 方式 4：Vue 组件扩展（推荐 ⭐⭐⭐⭐）

**适用场景**: 需要自定义 UI 的属性面板

```vue
<template>
  <div>
    <!-- hiprint 自带的属性 -->
    <div id="PrintElementOptionSetting"></div>
    
    <!-- 自定义扩展的属性 -->
    <div v-if="selectedElement">
      <a-divider>扩展属性</a-divider>
      <a-form-item label="我的属性">
        <a-input 
          v-model:value="customValue"
          @change="updateElement"
        />
      </a-form-item>
    </div>
  </div>
</template>

<script setup>
const selectedElement = ref(null);
const customValue = ref('');

const updateElement = () => {
  if (selectedElement.value) {
    selectedElement.value.options.customProp = customValue.value;
    selectedElement.value.refresh();
  }
};

// 监听元素选择
$(document).on('PrintElementSelectEvent', (e, element) => {
  selectedElement.value = element;
  customValue.value = element.options.customProp || '';
});
</script>
```

**优点**:
- ✅ UI 完全自定义
- ✅ 可以使用 Vue 生态
- ✅ 交互体验好

**缺点**:
- ❌ 需要手动同步数据
- ❌ 代码量较大

---

### 方式 5：程序化批量设置（推荐 ⭐⭐⭐）

**适用场景**: 批量设置多个元素的属性

```javascript
const batchSetProperties = (template, props) => {
  const elements = template.getPrintElements();
  
  elements.forEach(element => {
    // 根据类型设置不同属性
    if (element.printElementType.type === 'text') {
      Object.assign(element.options, props.text);
    } else if (element.printElementType.type === 'image') {
      Object.assign(element.options, props.image);
    }
  });
  
  template.update();
};

// 使用
batchSetProperties(hiprintTemplate, {
  text: { fontSize: 14, color: '#333' },
  image: { width: 100, height: 100 }
});
```

**优点**:
- ✅ 批量操作高效
- ✅ 适合主题切换

**缺点**:
- ❌ 需要遍历所有元素
- ❌ 可能影响性能

## 常见场景

### 场景 1：骑缝章切割配置

```javascript
// 1. 定义属性
window.HIPRINT_CONFIG = {
  optionItems: [
    {
      name: 'enableSplit',
      title: '启用切割',
      type: 'checkbox',
      defaultValue: false
    },
    {
      name: 'splitCount',
      title: '切割页数',
      type: 'number',
      defaultValue: 3,
      min: 2,
      max: 10
    },
    {
      name: 'overlapWidth',
      title: '重叠宽度',
      type: 'number',
      defaultValue: 10,
      min: 0,
      max: 50
    }
  ]
};

// 2. 使用属性
{
  tid: 'stamp.seamless',
  text: '骑缝章',
  type: 'image',
  options: {
    enableSplit: true,
    splitCount: 3,
    overlapWidth: 10,
    
    // 自定义渲染
    formatter: function(data, options) {
      if (options.enableSplit) {
        return `/api/stamp/split?count=${options.splitCount}`;
      }
      return data;
    }
  }
}
```

### 场景 2：条件显示

```javascript
{
  tid: 'conditional.text',
  text: '条件文本',
  type: 'text',
  options: {
    showInPage: 'first',  // 只在第一页显示
    
    formatter: function(data, options, currentPage, totalPages) {
      // 根据页码决定是否显示
      if (options.showInPage === 'first' && currentPage !== 0) {
        return '';
      }
      if (options.showInPage === 'last' && currentPage !== totalPages - 1) {
        return '';
      }
      return data;
    }
  }
}
```

### 场景 3：数据格式化

```javascript
{
  tid: 'format.text',
  text: '格式化文本',
  type: 'text',
  field: 'amount',
  options: {
    formatType: 'currency',  // 自定义属性
    prefix: '¥',
    suffix: '元',
    
    formatter: function(data, options) {
      let result = data;
      
      // 格式化为货币
      if (options.formatType === 'currency') {
        result = parseFloat(data).toFixed(2);
      }
      
      // 添加前缀后缀
      if (options.prefix) result = options.prefix + result;
      if (options.suffix) result = result + options.suffix;
      
      return result;
    }
  }
}
```

### 场景 4：样式联动

```javascript
window.HIPRINT_CONFIG = {
  optionItems: [
    {
      name: 'theme',
      title: '主题',
      type: 'select',
      options: [
        { value: 'light', label: '浅色' },
        { value: 'dark', label: '深色' }
      ],
      defaultValue: 'light',
      onChange: function(value, element) {
        // 主题改变时联动其他属性
        if (value === 'dark') {
          element.options.color = '#ffffff';
          element.options.backgroundColor = '#333333';
        } else {
          element.options.color = '#000000';
          element.options.backgroundColor = '#ffffff';
        }
        element.refresh();
      }
    }
  ]
};
```

## 最佳实践

### ✅ DO - 推荐做法

1. **使用语义化的属性名**
   ```javascript
   // ✅ 好
   { name: 'enableSplit', title: '启用切割' }
   
   // ❌ 差
   { name: 'prop1', title: '属性1' }
   ```

2. **提供默认值**
   ```javascript
   // ✅ 好
   {
     name: 'fontSize',
     type: 'number',
     defaultValue: 14  // 有默认值
   }
   ```

3. **添加验证**
   ```javascript
   // ✅ 好
   {
     name: 'url',
     type: 'input',
     validate: function(value) {
       return /^https?:\/\//.test(value);
     }
   }
   ```

4. **属性分组**
   ```javascript
   // ✅ 好 - 相关属性放在一起
   window.HIPRINT_CONFIG = {
     // 尺寸相关
     width: {},
     height: {},
     
     // 位置相关
     left: {},
     top: {},
     
     // 样式相关
     fontSize: {},
     color: {}
   };
   ```

5. **文档化**
   ```javascript
   // ✅ 好 - 添加注释
   {
     name: 'customProp',
     title: '自定义属性',
     type: 'input',
     // 说明：用于控制...
     defaultValue: ''
   }
   ```

### ❌ DON'T - 不推荐做法

1. **避免与内置属性冲突**
   ```javascript
   // ❌ 差 - 可能与内置属性冲突
   { name: 'width' }
   
   // ✅ 好 - 使用命名空间
   { name: 'customWidth' }
   ```

2. **不要在 formatter 中修改 options**
   ```javascript
   // ❌ 差
   formatter: function(data, options) {
     options.count++;  // 不要修改 options
     return data;
   }
   
   // ✅ 好
   formatter: function(data, options) {
     const count = options.count || 0;
     return `${data} (${count})`;
   }
   ```

3. **避免过多的属性项**
   ```javascript
   // ❌ 差 - 属性太多，用户体验差
   optionItems: [
     { name: 'prop1' },
     { name: 'prop2' },
     // ... 50 个属性
   ]
   
   // ✅ 好 - 分组或按需显示
   ```

## 故障排查

### 问题 1：属性不显示在面板上

**原因**:
- `settingContainer` 未正确绑定
- 属性未在 `supportOptions` 中声明
- `optionItems` 配置错误

**解决方案**:
```javascript
// 1. 确认容器绑定
const template = new window.hiprint.PrintTemplate({
  settingContainer: '#PrintElementOptionSetting'  // 确认 ID 正确
});

// 2. 在 supportOptions 中声明
window.HIPRINT_CONFIG = {
  text: {
    supportOptions: [
      { name: 'customProp' }  // 必须声明
    ]
  }
};

// 3. 检查 optionItems 配置
window.HIPRINT_CONFIG = {
  optionItems: [
    {
      name: 'customProp',  // 名称要匹配
      title: '自定义属性',
      type: 'input'
    }
  ]
};
```

### 问题 2：属性值不生效

**原因**:
- 属性值设置后未刷新
- formatter 函数未使用该属性

**解决方案**:
```javascript
// 1. 设置后刷新
element.options.customProp = 'newValue';
element.refresh();  // 必须刷新
hiprintTemplate.update();

// 2. 在 formatter 中使用
formatter: function(data, options) {
  // 使用自定义属性
  return options.customProp ? `${options.customProp}: ${data}` : data;
}
```

### 问题 3：属性变化不触发 onChange

**原因**:
- onChange 函数配置错误
- 事件未正确绑定

**解决方案**:
```javascript
{
  name: 'customProp',
  title: '自定义属性',
  type: 'input',
  onChange: function(value, element) {
    // 注意 this 指向
    console.log('新值:', value);
    console.log('元素:', element);
    
    // 触发刷新
    element.refresh();
  }
}
```

### 问题 4：自定义属性不保存

**原因**:
- 属性未在元素的 `options` 中
- 模板导出时丢失

**解决方案**:
```javascript
// 确保属性在 options 中
element.options.customProp = 'value';

// 导出时检查
const json = hiprintTemplate.getJson();
console.log('模板 JSON:', json);

// 导入时恢复
const template = new window.hiprint.PrintTemplate({
  template: json  // 自定义属性会自动恢复
});
```

## 总结

手动自定义属性的核心步骤：

1. **定义属性** → `window.HIPRINT_CONFIG.optionItems`
2. **绑定容器** → `settingContainer: '#PrintElementOptionSetting'`
3. **声明支持** → `supportOptions: [{ name: 'customProp' }]`
4. **使用属性** → 在元素的 `options` 和 `formatter` 中使用
5. **监听变化** → 通过 `onChange` 或 jQuery 事件监听

选择合适的方式：
- 📌 **需要 UI 显示** → 方式 1（全局配置）
- 📌 **元素特有属性** → 方式 2（元素定义）
- 📌 **自定义 UI** → 方式 4（Vue 组件）
- 📌 **批量操作** → 方式 5（程序化）

祝你使用愉快！🎉
